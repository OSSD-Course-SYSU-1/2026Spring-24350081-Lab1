

### 鸿蒙开发项目文件结构速查表

| 分类 | 目录/文件 | 类型 | 作用与大致内容说明 |
| :--- | :--- | :---: | :--- |
| **应用全局作用域** | `AppScope/` | 📂 目录 | **应用全局配置目录**，包含整个应用级别的公共配置和资源。 |
| | ├─ `app.json5` | 📄 文件 | **应用核心清单**，定义应用包名(bundleName)、版本号(version)、全局图标等。 |
| | └─ `resources/` | 📂 目录 | 存放应用级别的全局公共资源（如全局桌面图标）。 |
| **主入口模块** | `entry/` | 📂 目录 | **应用主模块(Module)**，编译后生成核心的 HAP 包，是应用的主体。 |
| | ├─ `src/` | 📂 目录 | 存放该模块的核心源代码（ArkTS界面逻辑、Ability、模块化资源等）。 |
| | ├─ `build/` | 📂 目录 | 编译构建的输出目录，包含编译过程产物和最终生成的 HAP 包（不提交Git）。 |
| | ├─ `build-profile.json5` | 📄 文件 | **模块级构建配置**，定义该模块的编译目标(targets)、架构等。 |
| | ├─ `hvigorfile.ts` | 📄 文件 | **模块级构建脚本**，定义该模块专属的 Hvigor 编译打包任务。 |
| | └─ `oh-package.json5` | 📄 文件 | **模块级依赖配置**，声明当前 entry 模块依赖的 ohpm 第三方包。 |
| **特色: AI/数据库** | `.lancedb/` | 📂 目录 | **本地向量数据库存储目录**，说明该应用集成了本地 AI 检索能力。 |
| | ├─ `knowledge_base.lance/`| 📂 目录 | 具体的 LanceDB 知识库向量数据表和索引文件。 |
| | └─ `ragConfig.json` | 📄 文件 | **RAG (检索增强生成) 配置文件**，配置大模型参数、Prompt、检索 Top-K 等。 |
| **工程根目录配置** | `build-profile.json5` | 📄 文件 | **工程级全局构建配置**，定义鸿蒙 SDK 版本、签名配置、包含哪些子模块。 |
| | `hvigorfile.ts` | 📄 文件 | **工程级全局构建脚本**，控制整个项目的 Hvigor 构建生命周期。 |
| | `oh-package.json5` | 📄 文件 | **工程级全局依赖配置**，声明整个工程共用的 ohpm 第三方包。 |
| **工程文档与测试** | `screenshots/` | 📂 目录 | 存放应用的展示截图（分中英文），通常用于 Readme 展示或上架应用市场。 |
| | `test/` | 📂 目录 | 工程级别的测试目录，通常用于存放端到端（E2E）集成测试代码。 |
| | `readme_cn.md` (及 en) | 📄 文件 | 项目的中文/英文说明文档（图标带 M↓ 表示本地有修改且有未拉取的更新）。 |
| | `LICENSE` | 📄 文件 | 项目的开源协议或版权声明文件。 |
| | `.gitignore` | 📄 文件 | Git 忽略规则文件，定义哪些文件（如 build, .idea）不上传到代码仓库。 |
| **IDE与构建系统** | `.idea/` | 📂 目录 | DevEco Studio (基于IDEA) 的工作区配置文件（UI状态、VCS绑定等）。 |
| | `.hvigor/` | 📂 目录 | Hvigor 鸿蒙构建工具在运行时的缓存目录。 |
| | `hvigor/` | 📂 目录 | 包含 `hvigor-config.json5`，用于配置构建工具本身的版本信息。 |
